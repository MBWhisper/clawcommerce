# دليل اختبار عملية التسجيل الشامل

## نظرة عامة على تدفق التسجيل

هذا الدليل يوضح خطوات اختبار النظام بشكل شامل من التسجيل إلى الوصول للداشبورد.

## المتطلبات قبل الاختبار

✅ **Supabase متصل بشكل صحيح**
- `VITE_SUPABASE_URL` مضبوطة
- `VITE_SUPABASE_ANON_KEY` مضبوطة

✅ **قاعدة البيانات جاهزة**
- جدول `profiles` موجود مع triggers
- جدول `stores` موجود
- جدول `subscriptions` موجود

✅ **البريد الإلكتروني** (اختياري للتطوير)
- يمكن تعطيل تأكيد البريد في Supabase للاختبار السريع

---

## خطوات الاختبار

### 1. اختبار التسجيل (حساب أول)

**البيانات:**
```
الاسم الكامل: محمد أحمد
البريد الإلكتروني: test1@example.com
كلمة المرور: TestPassword123
تأكيد المرور: TestPassword123
```

**الخطوات:**
1. اذهب إلى `/register`
2. أدخل البيانات أعلاه
3. اقرأ وافق على الشروط
4. اضغط "إنشاء حساب مجاني"

**الفحوصات:**
- ✓ لا تظهر رسائل خطأ
- ✓ يتم إنشاء المتجر افتراضي (test1-Store)
- ✓ ينتقل للداشبورد مباشرة
- ✓ يظهر الاسم في الداشبورد "مرحباً، محمد أحمد"

---

### 2. اختبار تسجيل الدخول

**البيانات:**
```
البريد الإلكتروني: test1@example.com
كلمة المرور: TestPassword123
```

**الخطوات:**
1. اذهب إلى الداشبورد أو اضغط logout
2. اذهب إلى `/login`
3. أدخل البيانات
4. اضغط "تسجيل الدخول"

**الفحوصات:**
- ✓ تسجيل الدخول ينجح
- ✓ يظهر اسم المستخدم في الداشبورد
- ✓ البيانات محفوظة من التسجيل السابق

---

### 3. اختبار التسجيل (حساب ثاني)

**البيانات:**
```
الاسم الكامل: فاطمة علي
البريد الإلكتروني: test2@example.com
كلمة المرور: SecurePassword456
تأكيد المرور: SecurePassword456
```

**الخطوات:**
1. تسجيل خروج من الحساب الأول (اضغط على زر الخروج)
2. اذهب إلى `/register` مجدداً
3. أدخل بيانات مختلفة
4. أكمل عملية التسجيل

**الفحوصات:**
- ✓ ينجح إنشاء حساب جديد
- ✓ متجر جديد يُنشأ تلقائياً
- ✓ البيانات منفصلة عن الحساب الأول
- ✓ يظهر "مرحباً، فاطمة علي"

---

### 4. اختبار التبديل بين الحسابات

**الخطوات:**
1. تسجيل خروج (logout)
2. تسجيل دخول بـ test1@example.com
3. تحقق من البيانات (محمد أحمد)
4. تسجيل خروج
5. تسجيل دخول بـ test2@example.com
6. تحقق من البيانات (فاطمة علي)

**الفحوصات:**
- ✓ كل حساب يحتفظ ببيانات منفصلة
- ✓ الاسم يتغير حسب الحساب
- ✓ المتجر يختلف لكل حساب

---

### 5. اختبار الداشبورد

عند الدخول، تحقق من:

**القسم العلوي (Welcome Section):**
- ✓ اسم المستخدم صحيح
- ✓ الخطة الحالية تظهر (مجانية)

**الإحصائيات:**
- ✓ إجمالي الإيرادات = 0 (محفظة فارغة)
- ✓ عدد الطلبات = 0
- ✓ عدد المنتجات = 0
- ✓ عدد العملاء = 0

**جدول الطلبات الأخيرة:**
- ✓ رسالة "لا توجد طلبات بعد" تظهر

**أفضل المنتجات:**
- ✓ رسالة "لا توجد منتجات بعد" تظهر

**الإجراءات السريعة:**
- ✓ ثلاثة أزرار متوفرة:
  - إضافة منتج جديد
  - إنشاء طلب يدوي
  - إنشاء كوبون خصم

---

### 6. اختبار الأخطاء الشائعة

#### تكرار البريد الإلكتروني
```
1. حاول التسجيل بـ test1@example.com مرة أخرى
✓ رسالة: "هذا البريد الإلكتروني مسجل بالفعل"
```

#### كلمة مرور ضعيفة
```
1. أدخل كلمة مرور: "123"
✓ رسالة: "كلمة المرور يجب أن تكون 6 أحرف على الأقل"
```

#### عدم تطابق كلمات المرور
```
1. أدخل: TestPassword123
2. تأكيد: DifferentPassword456
✓ رسالة: "كلمتا المرور غير متطابقتين"
```

#### بريد إلكتروني غير صحيح
```
1. أدخل: notanemail
✓ رسالة: "البريد الإلكتروني غير صحيح"
```

## استكشاف الأخطاء

### مشكلة: "فشل إنشاء الحساب"
**الحل:**
1. تحقق من Supabase env variables
2. تحقق من جدول profiles موجود
3. اقرأ console logs للتفاصيل

### مشكلة: لا يتم الانتقال للداشبورد
**الحل:**
1. جرب تحديث الصفحة
2. تحقق من localStorage
3. اقرأ console errors

### مشكلة: الداشبورد فارغ
**الحل:**
1. تحقق من createStore trigger
2. تحقق من RLS policies
3. تحقق من store_id صحيح

## معلومات مفيدة

**URLs الرئيسية:**
- الرئيسية: `/`
- التسجيل: `/register`
- تسجيل الدخول: `/login`
- الداشبورد: `/dashboard`
- نسيت كلمة المرور: `/forgot-password`

**Console Debugging:**
```javascript
// تحقق من حالة المستخدم
useAuthStore.getState().user

// تحقق من المتجر
useAuthStore.getState().store

// تحقق من الملف الشخصي
useAuthStore.getState().profile
```
